# PRD.md — Student Career Plan Platform (UK) v4 (Aligned)

---

# 1. EXECUTIVE SUMMARY

A digital version of the existing “My Career Plan” booklet that:
- Preserves structure, language, and flow
- Adds simple digital interaction, evidence capture, and reporting
- Supports SEN students with low-friction UX
- Produces Ofsted-ready outputs automatically

Principle:
> Mirror the booklet first. Enhance second.

---

# 2. CORE STRUCTURE (1:1 WITH BOOKLET)

The system MUST follow this exact hierarchy:

For EACH:
GB1 → GB8

Each contains:

1. Benchmark explanation  
2. “What I need to do” (activities list)  
3. Tick (Done)  
4. Date  
5. ASDAN unit reference  
6. Reflection question (fixed wording)

---

# 3. USER EXPERIENCE

## Student
- Sees GB1–GB8 as sections
- Each section = checklist
- Tap “Done”
- Add photo OR voice reflection
- See progress bar

## Staff
- Validate completion (optional)
- Add notes quickly
- Run term review

## Leadership
- Dashboard view of:
  - Completion
  - Engagement
  - Evidence

---

# 4. FEATURE SET (STRICTLY ALIGNED)

## 4.1 Benchmark Pages (GB1–GB8)

Each page includes:
- Fixed benchmark description
- “What I need to do” list
- Tick + date per item
- ASDAN mapping visible
- Reflection prompt (exact wording)

---

## 4.2 Activity Completion

Default interaction:
- Tap → Done
- Auto date recorded

Optional:
- Add note
- Add evidence

---

## 4.3 Evidence Capture (Mobile First)

- Photo (primary)
- Video
- File

Offline capable:
- Store locally
- Sync when online

---

## 4.4 Voice Reflection (MVP)

- Tap → Speak
- Stored + transcribed
- Linked to benchmark

---

## 4.5 SMART Targets (GB8)

- 3 targets minimum
- Fields:
  - Target
  - Steps
  - Deadline
  - Done

---

## 4.6 Employer Encounter Log

Fields:
- Date
- Employer
- Sector
- What I learned

Progress indicator:
- e.g. 2/3 completed

---

## 4.7 Tutor / Adviser Review Record

Structured (locked format):
- Date
- Reviewer
- Notes
- Next steps
- Signature

---

## 4.8 ASDAN Summary

- Unit
- Level
- Date
- Signature

Always visible, even if optional

---

## 4.9 External Profile

- Generated from:
  - Completed benchmarks
  - Evidence
  - Targets

Controlled by:
- Staff approval required

---

# 5. DASHBOARD (ADDED LAYER)

Top-level view:

- GB1–GB8 progress bars
- “What’s next” prompts
- Missing items highlighted
- Employer encounters count
- Targets status

---

# 6. SEN DESIGN RULES

- One action per screen
- Minimal typing
- Large buttons
- Consistent layout
- Photo + voice first

# 7. STAFF FEATURES
